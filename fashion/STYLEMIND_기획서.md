## 1. 프로젝트 주제

### 1.1 프로젝트명

**STYLEMIND/코디라이프** - 멀티 에이전트 기반 개인화 패션 추천 시스템

### 1.2 프로젝트 정의

사용자의 대화 입력을 기반으로 날씨, 상황(TPO), 개인 옷장, 스타일 선호, 외부 쇼핑 데이터 등을 종합하여 개인화된 패션 추천을 생성하는 지능형 멀티 에이전트 추천 시스템

### 1.3 핵심 키워드

- **멀티 에이전트 아키텍처**: 역할 분리된 전문 에이전트 협업
- **LangGraph 기반 Orchestration**: 사고 흐름 그래프 구현
- **상태 기반 추론**: 공용 State를 통한 맥락 누적 및 재검증
- **개인화 추천**: TPO, 날씨, 옷장, 체형, 취향 종합 분석

## 2. 문제 정의

### 2.1 우리가 해결하려는 문제: "Show" vs "Decide"

기존 패션 서비스들은 **"이 옷이 나한테 어울려 보이는가?"**(시각적 매칭)에 집중합니다. 가상 피팅(VTO)이 대표적입니다. 그러나 정작 사용자가 매일 아침 고민하는 질문은 다릅니다: **"오늘 이 옷을 입어도 되는가?"**(상황적 적합성)

VTO로 정장을 입혀봤는데 핏이 완벽했습니다. 하지만 그날이 **야유회**라면 그 선택은 틀린 것입니다. 기존 기술은 '시각적 핏'은 보여주지만, '상황적 핏(TPO)'은 판단하지 못합니다. **우리는 그 빈틈을 파고들었습니다.**

### 2.2 기존 서비스의 한계: 맥락 인식 부재

#### 맥락 없는 시각화

현재 상용화된 가상 피팅(VTO) 서비스들은 사용자가 선택한 옷을 시각적으로 입혀주는 데 탁월합니다. 그러나 "오늘 날씨에 이 소재가 적합한지", "이 모임 성격에 이 스타일이 맞는지"와 같은 비시각적 맥락은 전혀 고려하지 못합니다.

#### 단편적 필터링

기존 쇼핑몰은 '카테고리/색상' 등 단순 속성 필터링만 제공합니다. "비 오는 날 야외 데이트"처럼 복합적인 조건에 맞는 상품을 추천받으려면, 사용자가 직접 날씨를 확인하고 여러 번 검색해야 하는 번거로움이 있습니다.

### 2.3 본 프로젝트의 접근 방식

본 시스템은 이러한 복합적 판단을 하나의 거대한 프롬프트로 처리하지 않고, 역할이 분리된 여러 전문 에이전트들이 협업하는 구조로 모델링합니다. 각 에이전트가 날씨, TPO, 옷장, 스타일 평가를 분담하고, 중앙 Orchestrator가 이를 조율하여 최종 추천을 도출합니다.

## 3. 시스템 구성 기획

### 3.1 멀티 에이전트 구성

각 에이전트는 다음과 같은 전문 역할을 담당합니다.

| 에이전트 | 역할 |
|----------|------|
| **WeatherAgent** | 날씨 및 환경 맥락 해석 (온도, 습도, 강수 확률 등) |
| **TPOAgent** | 사용 목적 및 상황 추론 (업무, 데이트, 캐주얼 등) |
| **WardrobeAgent** | 개인 옷장 및 과거 착용 이력 분석 |
| **StyleEvalAgent** | 코디 조합의 적합성 평가 및 점수화 |
| **ProductAgent** | 상품 메타데이터 수집 및 정리 |
| **WebSearchAgent** | 쇼핑 사이트 검색, 외부 상품 정보 수집(최신정보) |
| **LLMGenerator** | 수집된 맥락과 추천 결과를 바탕으로 자연어 응답 생성 |

### 3.2 LangGraph 기반 Orchestrator

이들을 지휘하는 중앙 Orchestrator는 단순한 호출 스크립트가 아니라, 분기, 조건 판단, 반복 검증이 가능한 사고 흐름 그래프(LangGraph)로 구현됩니다.

모든 에이전트는 하나의 공용 작업 메모리(State)를 공유하며, 각 단계의 중간 결과(날씨 요약, 후보 아이템, 평가 점수, 검증 결과 등)는 이 State에 누적되고 다음 단계의 판단 근거로 사용됩니다.

## 4. 핵심 특징

### 4.1 설명 가능한 추천 (Explainable Recommendation)

단순히 상품 링크를 나열하는 것이 아니라, **추천의 논리적 근거(Why)**를 함께 제시합니다. 개인 프로필을 바탕으로 WeatherAgent와 TPOAgent의 분석 결과를 바탕으로 **"오후에 습도가 높으니 린넨 소재를 추천합니다"**와 같이 사용자를 설득할 수 있는 맥락을 제공합니다. 단순 쇼핑몰 검색과 퍼스널 스타일리스트의 차이입니다.

### 4.2 에이전트 협업 지능 (Agentic Orchestration)

옷장 데이터(정적), 실시간 날씨(동적), 쇼핑몰 트렌드(외부)라는 성격이 전혀 다른 3가지 데이터 소스를 LangGraph 상에서 실시간으로 결합합니다.

단일 모델의 한 번 추론으로는 불가능한, 다단계 검증과 재탐색을 통한 초개인화된 의사결정을 수행합니다.

### 4.3 실시간 대화형 경험

텍스트와 메타데이터 기반의 빠른 추론을 통해, 사용자와 실시간으로 티키타카가 가능한 '채팅형 코디네이터' 경험을 구현합니다.

**"비 오는 날 흰 바지는 피하세요"**라고 즉시 조언해주는 지능적 판단이, 화려한 시각 효과보다 사용자에게 더 실질적인 도움을 줍니다.

### 4.4 개인화 요소

- **날씨 맥락** - 실시간 기온, 체감온도, 강수 확률 반영
- **TPO 분석** - 목적, 장소, 시간대에 따른 적합성 판단
- **개인 옷장** - 보유 의류 데이터베이스 및 착용 이력
- **체형 및 취향** - 사용자 프로필 기반 스타일 선호도
- **외부 트렌드** - 쇼핑몰 신상품 및 인기 아이템 연동
